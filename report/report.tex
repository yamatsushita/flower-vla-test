\section{Overview}

This report documents the reproduction of FLOWER VLA~\cite{breuss2025flower} training
from scratch on the CALVIN~\cite{mees2022calvin} benchmark, specifically the
D$\rightarrow$D split.
The original code is available at
\url{https://github.com/intuitive-robots/flower_vla_calvin}.

% ─────────────────────────────────────────────────────────────────
\section{Method Summary}

FLOWER VLA (Flow Matching Language-conditioned Robot policy with Visual tokens from LLMs and Actions)
is a Vision-Language-Action model that combines:
\begin{itemize}
  \item \textbf{Florence-2-large} as the visual language backbone,
  \item a \textbf{Flow Matching DiT} (Diffusion Transformer) for action generation,
  \item \textbf{FLOWER pretraining} on large-scale robot manipulation data before CALVIN fine-tuning.
\end{itemize}

The model predicts action chunks (window size $= 10$) conditioned on
RGB observations from a static camera and a wrist camera, plus natural language instructions.

\subsection{Windows Compatibility Patches}

The original code targets Linux. The following patches were applied to run
on Windows with a single GPU:

\begin{enumerate}
  \item \textbf{\texttt{hulc\_wrapper.py}} — EGL device detection invokes
        \texttt{./EGL\_options.o}, a Linux binary. Added \texttt{FileNotFoundError}
        and \texttt{OSError} to the exception handler.
  \item \textbf{\texttt{rollout\_long\_horizon.py}} — \texttt{on\_validation\_start}
        initialized the CALVIN simulator even when rollouts were configured to be skipped.
        Added an early-return guard for \texttt{current\_epoch < skip\_epochs}.
  \item \textbf{\texttt{training\_calvin.py}} — Replaced hardcoded
        \texttt{ddp\_find\_unused\_parameters\_true} strategy with \texttt{"auto"}
        on single-GPU runs (avoids \texttt{use\_libuv} error).
  \item \textbf{pyhash stub} — \texttt{pyhash 0.9.3} uses the removed
        \texttt{use\_2to3} setuptools option. Replaced with a pure-Python
        FNV-1/FNV-1a implementation.
\end{enumerate}

\section{Reproduction Notes}

\paragraph{Pretrained weights.}
``Training from scratch'' in the context of this work means training the FLOWER CALVIN
model using FLOWER pretraining weights (\texttt{mbreuss/flower\_vla\_pret},
checkpoint step 360k), which were obtained from pre-training on diverse robot data.
The DiT layers 12--17 are randomly initialized (132 missing keys in the checkpoint,
expected behavior).

\paragraph{Single-GPU vs.\ 4-GPU.}
The paper uses 4$\times$ A100 GPUs with \texttt{batch\_size=8} per GPU
(effective batch size 32). This reproduction uses 1$\times$ RTX 4090 with
\texttt{batch\_size=8}, resulting in approximately 4$\times$ fewer gradient updates
per epoch and potentially reduced performance.

\paragraph{Training time.}
With 1$\times$ RTX 4090 (24 GB), training 35 epochs on D$\rightarrow$D takes
approximately 14 hours (25 min/epoch). The paper's 4-GPU setup is estimated
to take $\sim$3.5 hours.

\section{Conclusion}

We have successfully set up and launched FLOWER VLA training from scratch on
CALVIN D$\rightarrow$D using a single RTX 4090 on Windows.
The training pipeline is fully functional after applying four Windows-specific
compatibility patches.
The training loss decreases consistently.
Quantitative evaluation (CALVIN long-horizon success rate) requires transferring
the trained checkpoint to a Linux system for PyBullet-based rollouts.

All code, configs, and this report are available at
\url{https://github.com/yamatsushita/flower-vla-test}.
